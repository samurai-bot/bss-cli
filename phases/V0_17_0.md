# v0.17.0 — Telco Hygiene (MNP, MSISDN replenishment, Roaming)

> **The release that closes three real-telco gaps any operator would call out on a first walkthrough.** v0.17 is intentionally narrow: no new doctrine pillars, no new auth model, no new external integrations. One Alembic migration, three feature areas, and a small set of operator-only surfaces. After v0.16's payment build-out, this is the "tidy up" release.

## Goal

Three deliverables, scoped to a ~2–3 day window:

1. **MNP (port-in / port-out)** — operator-driven `crm.port_request` aggregate with FSM `requested → validated → completed | rejected`. No customer self-serve. Cockpit `/ports` slash command + `port_request.*` tools approve/reject. Port-in seeds the donor MSISDN into the pool; port-out flips the MSISDN to terminal `ported_out` and terminates the target subscription with `release_inventory=False`.
2. **MSISDN replenishment** — `bss inventory msisdn add-range <prefix> <count>` (CLI + cockpit tool wrapper) bulk-extends the pool. Idempotent on overlap. Plus a low-watermark `inventory.msisdn.pool_low` event when available count drops to or below `BSS_INVENTORY_MSISDN_POOL_LOW_THRESHOLD` (default 50).
3. **Roaming** — new `data_roaming` allowance type wired through catalog → subscription → mediation → rating → portal. Plans get bundled roaming MB (PLAN_S=0, PLAN_M=500, PLAN_L=2048). New `VAS_ROAMING_1GB` top-up. `UsageEvent.roaming_indicator: bool` routes the decrement to the roaming balance. Block-on-exhaust applies independently — exhausting roaming does NOT block the subscription itself.

When v0.17 ships, the BSS-CLI plan model includes roaming, the operator can recover from MSISDN-pool exhaustion, and a customer's number can move in or out of the network — all without editing the doctrine of motto #1 (bundled prepaid only) or #3 (block-on-exhaust).

## Schema delta — single migration `0019`

`packages/bss-models/alembic/versions/0019_v0170_mnp_roaming_replenishment.py` (down_revision="0018"):

- **`crm.port_request`** — id (PORT-NNN), direction (`port_in|port_out`), donor_carrier, donor_msisdn, target_subscription_id (NULL ok for port-in), requested_port_date, state (default `requested`), rejection_reason. CHECK constraints on direction + state. Partial unique index on `(donor_msisdn, tenant_id) WHERE state IN ('requested','validated')` mirrors the policy `port_request.create.donor_msisdn_unique_among_pending`. Secondary index on `(state, direction)` for the `/ports` list page.
- **`mediation.usage_event.roaming_indicator`** — `BOOLEAN NOT NULL DEFAULT FALSE`. Server default critical: every existing scenario YAML and rating fixture posts usage without the field.

Two non-DDL changes share the migration's release scope:

- **`inventory.msisdn_pool.status='ported_out'`** — application-level FSM only (the column is plain Text by design). `MsisdnRepository.mark_ported_out` sets `quarantine_until='9999-12-31'` so reserve-next predicates skip the row by construction.
- **`bundle_allowance.allowance_type='data_roaming'`** — column is open Text; v0.17 introduces the value via seeds.

## What ships

### Catalog seeds (`packages/bss-seed/bss_seed/catalog.py`)

- New `BundleAllowance` rows: `BA_S_ROAM` (PLAN_S, 0 mb), `BA_M_ROAM` (PLAN_M, 500 mb), `BA_L_ROAM` (PLAN_L, 2048 mb). Zero on PLAN_S so the bar renders absent (filtered in `_line_view`) and the snapshot doctrine remains intact.
- New `VasOffering`: `VAS_ROAMING_1GB`, 8.00 SGD, allowance_type `data_roaming`, qty 1024 mb, no expiry.

### Subscription wiring (zero new code in `create()`)

`subscription_service.create()` already iterates `offering.bundleAllowance` and creates a `BundleBalance` per allowance. New `data_roaming` rows materialize automatically at signup.

The non-trivial change is in `purchase_vas`: when no balance row exists for the VAS allowance type, it now *creates* the row from the VAS spec. Previously the no-row branch was a silent no-op (safe for data/voice/sms because those rows always exist post-create). Generic fix — unblocks any future allowance type the same way.

### Mediation (`services/mediation/`)

`UsageCreateRequest` gains `roaming_indicator: bool = False`. The field flows: API route → service kwarg → `UsageEvent.roaming_indicator` column → `to_payload` adds `"roamingIndicator"` to the published `usage.recorded` event.

### Rating (`services/rating/app/events/consumer.py`)

The pure `rate_usage` function stays unchanged. Routing happens *after* rate_usage returns: if `roamingIndicator=true` and `result.allowance_type == "data"`, validate the offering carries `data_roaming` then override the published `allowanceType` to `data_roaming`. If the offering has no roaming allowance, publish `usage.rejected` with reason `rating.no_roaming_allowance` and return.

This split — pure function unchanged + routing in the consumer — keeps every existing rating fixture and unit test working without modification.

### Subscription block-on-exhaust independence

`is_exhausted` already uses `primary_type='data'` by default, so `data_roaming` exhaustion does not cascade. `handle_usage_rated` now invokes a new policy `subscription.usage_rated.roaming_balance_required` for any `data_roaming` decrement: missing row OR exhausted balance → `usage.rejected` event, subscription stays `active`, home data unaffected.

### MSISDN replenishment

- `MsisdnRepository.add_range(prefix, count, tenant)` — bulk INSERT … ON CONFLICT DO NOTHING, returns `(inserted, skipped)`.
- `MsisdnRepository.count_available(tenant)` — used by the low-watermark hook.
- `MsisdnRepository.mark_ported_out(msisdn, subscription_id)` — terminal status with `quarantine_until='9999-12-31'`.
- New policies `msisdn.add_range.sane_prefix` (4–7 digits, count 1..10000) + `msisdn.release.terminal_status` (rejects release of `ported_out`).
- `InventoryService.add_msisdn_range(...)` emits `inventory.msisdn.range_added`.
- `InventoryService._maybe_emit_pool_low(tenant)` runs post-commit on every successful reserve; emits `inventory.msisdn.pool_low` when available drops below threshold. Errors swallowed — observability nudge, not a transactional invariant.
- HTTP route `POST /inventory-api/v1/msisdn/add-range`. CLI `bss inventory msisdn add-range <prefix> <count>`. Cockpit tool `inventory.msisdn.add_range` (operator_cockpit profile only).

### Port request entity + flows

- `bss_models.crm.PortRequest` ORM model + CRM repository + `port_request_state` FSM helper + `port_request` policies module.
- `PortRequestService.create / approve / reject`. Approve dispatches:
  - **Port-in**: insert/upsert donor MSISDN into `inventory.msisdn_pool` (status `assigned` if target_subscription_id set, else `available`). Emits `port_request.approved`, `inventory.msisdn.seeded_from_port_in`, `port_request.completed`.
  - **Port-out**: `MsisdnRepository.mark_ported_out` flips status + sets quarantine. `SubscriptionClient.terminate(release_inventory=False)` skips the MSISDN release path (eSIM still recycles). Emits `port_request.approved`, `inventory.msisdn.ported_out`, `port_request.completed`.
- HTTP routes `GET / POST /port-requests`, `POST /port-requests/{id}/approve`, `POST /port-requests/{id}/reject`. Mounted under `/crm-api/v1`.
- Subscription-service signature change: `terminate()` gains `release_inventory: bool = True` kwarg. Plumbed through API + `SubscriptionClient.terminate`.
- Cockpit tools: `port_request.list / get / create / approve / reject`. Operator_cockpit profile only — `validate_profiles()` self-checks.
- REPL slash command `/ports` (list / approve PORT-NNN / reject PORT-NNN <reason>) — mirrors the `/360` shape and persists turns to the cockpit Conversation store.

### Portal display (`portals/self-serve/`)

- `landing.py:ALLOWANCE_LABEL` adds `"data_roaming": "Roaming"`.
- `_line_view` filters the bars list so the Roaming bar is hidden when `total == 0 and remaining == 0` (PLAN_S users with no top-up don't see a stranded "Roaming 0/0" bar).
- `templates/partials/line_card.html` — no template change. Existing `{% for bar in line.bars %}` loop renders any number of bars.
- VAS_ROAMING_1GB picks up automatically on `/top-up` (catalog-driven). Step-up gate is the existing `vas_purchase` label.

## Doctrine amendments (CLAUDE.md)

Five v0.17+ anti-patterns appended:

- Don't overload `Case` for port requests — `crm.port_request` is its own aggregate with its own FSM and audit-event family.
- Don't release a ported-out MSISDN back to `available` — `ported_out` is terminal; recycling re-issues a number we've handed away.
- Don't add a new `event_type` for roaming — it's a per-event `roaming_indicator` attribute, not an event-type discriminator. `VALID_EVENT_TYPES` stays at `{data, voice, voice_minutes, sms}`.
- Don't cascade roaming-balance exhaustion into home-data exhaustion — `data_roaming` is *additive*; running out blocks roaming usage but does not block the subscription.
- Don't expose port-request writes to `customer_self_serve` — MNP is operator-driven by spec.

## Doctrine grep guards (Makefile `doctrine-check`)

Two new gates:

- `data_roaming` must NOT appear in `services/rating/app/domain/rating.py` (proves routing lives in the consumer, not the pure function).
- No code path in `services/crm/app/repositories/msisdn_repo.py` may flip a `ported_out` row back to `available`.

The cockpit-profile guard for port_request lives in `validate_profiles()` itself (runs at orchestrator import time, fails the deploy on drift).

## Known limits (intentional)

- **No automated donor-carrier validation.** The `validated` FSM state is a hook; v0.17 ships only the operator-driven `requested → completed` path. A future automated check would consume a `port_request.created` event and transition to `validated` (or to `rejected` with a structured reason).
- **No inbound port pre-validation flow** (the customer-facing "is my number portable?" check). Channel-layer concern; same posture as eKYC.
- **No per-country roaming tariffs.** Roaming is a single bucket — `data_roaming` MB. Per-country pricing would require `serving_plmn` on `UsageEvent` plus a tariff lookup; explicitly out of scope for v0.17 ("we offer just data roaming, because that's the only thing that matters").
- **No SM-DP+ rearm for ported-out numbers.** Port-out terminates the subscription (eSIM recycles); the GSMA SGP.22 rearm flow remains a post-v0.1 SOM task as documented in `CLAUDE.md`.

## Rollout

1. Bump `BSS_RELEASE = "0.17.0"` (already in this branch).
2. `make migrate` — applies `0019`. Backwards-compat is the column server defaults; pre-v0.17 rows keep working.
3. `make seed` — picks up the new BundleAllowance rows and `VAS_ROAMING_1GB`.
4. Recreate the affected service containers so they reload env (`BSS_INVENTORY_MSISDN_POOL_LOW_THRESHOLD`):
   ```
   docker compose up -d --force-recreate crm subscription mediation rating portal-self-serve portal-csr
   ```
5. Run `make scenarios-hero` against the running stack.
